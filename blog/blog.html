Q1. Discuss the scope of var, let, and const

Ans:
var: The var is the oldest keyword to declare a variable in JavaScript.

Global scoped or function scoped. The scope of the var keyword is the global or function scope. It means variables defined outside the function can be accessed globally, and variables defined inside a particular function can be accessed within the function.

Ans:
let: The let keyword is an improved version of the var keyword.

The scope of a let variable is only block scoped. It can’t be accessible outside the particular block ({block}).

Ans:
Const: The const keyword has all the properties that are the same as the let keyword, except the user cannot update it.

When users declare a const variable, they need to initialize it, otherwise, it returns an error. The user cannot update the const variable once it is declared.

Q2. Tell us the use cases of null and undefined

Ans :

Null

  • Null: It is the intentional absence of the value. It is one of the primitive values of JavaScript.
  • Explicitly assigned: null is a value that can be assigned to a variable or property to indicate that it intentionally has no value or is empty.

Undefined

  • Undefined: It means the value does not exist in the compiler. It is the global object.
  • Default value: Variables that are declared but not initialized, function parameters that are missing arguments, and object properties that don't exist return undefined by default.

Q3. What do you mean by REST API?

Ans: Representational State Transfer (REST) is an architectural style that defines a set of constraints to be used for creating web services. REST API is a way of accessing web services in a simple and flexible way without having any processing.
REST technology is generally preferred to the more robust Simple Object Access Protocol (SOAP) technology because REST uses less bandwidth, simple and flexible making it more suitable for internet usage. It’s used to fetch or give some information from a web service. All communication done via REST API uses only HTTP request.
